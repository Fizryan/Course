JobWave (Job Finder Application Design)

Description: A design project for a web and mobile application aimed at helping users find job opportunities. The application is designed using Figma to ensure an intuitive and user-friendly interface.

Course: Design Thinking

Objectives:

  • To design a user-friendly interface that enhances the job search experience.
  • To facilitate easy navigation and interaction within the application.

Process: The project began with brainstorming sessions to identify user pain points. We created wireframes in Figma and iterated on designs based on user feedback, resulting in high-fidelity prototypes that captured the intended user experience.

Features:

  • User profiles for personalized job recommendations
  • Application tracking system for users to manage their job applications
  • Responsive design for both web and mobile platforms
  • Interactive mockups created in Figma for user testing and feedback

Technologies: Figma (for design)

Challenges: The main challenge was ensuring a consistent user experience across both web and mobile platforms. Achieving a responsive design that maintained the same level of functionality and aesthetics on different screen sizes required several iterations. Additionally, gathering and incorporating user feedback from multiple rounds of testing posed difficulties in prioritizing features and changes.

Reflection: This project emphasized the importance of user-centered design and iterative feedback in the development process. It enhanced my design skills and my understanding of how to effectively communicate ideas visually.

Description: A movie ticketing system built in Go that allows users to view, search, and purchase movie tickets. Administrators can manage movie listings by adding, editing, and deleting entries.

  • Supports movie schedule and pricing management

Challenges: One of the biggest challenges was managing the complexity of the ticketing system‚Äôs backend.
